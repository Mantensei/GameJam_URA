# GAME_DESIGN — 「裏メニューを食べたい！」

## 概要
Unity 2022.3.11f1 製の GameJam プロジェクト（WebGLビルド）。
常連客を観察し、裏メニューを注文するために必要な行動を模倣する推理アクションゲーム。

- ジャンル：観察→模倣→推理（リアルタイム進行）
- 視点：2Dサイドビュー
- ビジュアル：ドット絵
- クリア条件：裏メニューを注文する
- 敗北条件：所持金が0になる

---

## コアループ

```
来店 → 観察＆模倣（リアルタイム同時進行） → 店主に話しかける → 判定 → クリア or 再来店（所持金不足でゲームオーバー）
```

1来店の中にフェーズ区切りはない。プレイヤーも客も自由に行動する。
時間はリアルタイムで進み、客は自身のタイムラインに沿って行動する。
1来店あたり2〜3分程度を想定。時間制限はない。

---

## 操作

ボタン4つ：

| ボタン | 用途 |
|--------|------|
| 左右移動 | 店内を移動、客や物に近づく |
| ジャンプ | 入店方法等（要検証、不要なら削除） |
| 定型文 | 感想などの発言系アクション（いつでも自由に使用可能） |
| インタラクト | 対象に応じた行動（席に座る/立つ、メニューを開く、食べる、店主に話しかける） |

---

## プレイヤーの行動フロー

全て任意・自由行動。最低限は「入店→店主に話しかける→退店」でも成立する。

```
入店 → 自由移動 → 席にインタラクト（座る） → メニューを開く → 注文（即支払い）
→ 料理待ち（離席自由） → 料理にインタラクト（食べる） → 感想（任意・いつでも）
→ 席にインタラクト（立つ） → 自由移動 → 店主に話しかける → 判定フェーズ → 自動退店
```

- 席に座る/立つはインタラクトで行う
- 注文時に即座に食事代が所持金から引かれる
- 料理は注文後しばらくしてから届く（待ち時間あり）
- 料理待ちの間も自由に離席・移動可能
- 食べるのもインタラクトで行う
- 感想（定型文）はいつでも自由に言える
- 店主に話しかけると判定フェーズに入り、その後自動退店

---

## ノルマ・ドボン・判定

### ノルマ
- ステージごとに設定された「裏メニューを頼むための条件」
- 全て達成すると店主が裏メニューを教えてくれる
- 難易度が高いほど条件が増える（一部は達成しなくてもOKな場合あり）

### ドボン
- 実行すると罰金等のペナルティが発生する禁止行動
- 常連客がやらない行動を避ければ基本的には踏まない
- 難易度が高いほど増える
- 判定は退店時にまとめて行う

### 判定（店主に話しかけた時）
- 全ノルマ達成 → 「裏メニューを教えます」 → クリア
- 未達成 / 何もしていない → 「帰んなさい」 → 自動退店
- ドボンがあった場合は退店時に罰金が引かれる

---

## 店内構成

- 舞台は定食屋
- カウンター席のみ（横並び）
- 店主はカウンターの向こう側

---

## 客

### 常連客
- 裏メニューを知っている客
- 食後に店主にコソコソ声で裏メニューを追加注文する → 常連の識別手段
- 行動の中にノルマのヒントが含まれる
- ただし常連の行動が全てノルマとは限らない（ダミーあり）
- Max 4人

### 一般客
- メニュー表にある品だけを注文する
- 行動にはドボンに繋がるものが含まれる場合がある

### 行動タイムライン
全ての客は以下の流れで行動する（リアルタイム進行）：

```
入店 → 着席 → 注文 → 食事 → 感想 → 店主にコソコソ話（常連のみ・裏メニュー追加注文） → 退店
```

---

## 盗み聞き

- 常連は食後に店主にコソコソ声で裏メニューを追加注文する
- プレイヤーが席に座っていれば、コソコソ話の内容（裏メニュー名）が聞こえる
- 座っていなければ聞こえない
- 盗み聞きした裏メニュー名は直接表示される

---

## メニュー・感想

### メニュー
- メニュー表に載っている品 / 載っていない品（裏メニュー）がある
- 複数品注文可能（合計金額が食事代）
- メニューUIを開くと画面を占有する → 開いている間も客の行動は進む → 観察とのトレードオフ

### 感想（定型文）
- メニューとは完全に独立した汎用表現（組み合わせ爆発を避ける）
- いつでも自由に使用可能
- 例：「うまい」「量が多い」「また来る」「まずい」「おかわり」

---

## コスト構造

| 項目 | 内容 |
|------|------|
| 所持金 | ステージごとに初期値が決まる |
| 食事代 | 注文時に即引き。注文した品数に応じて変動（単価はメニューごと） |
| ドボンペナルティ | 退店時に罰金（金額はドボン行動による） |

所持金が尽きるまでの来店回数が試行回数の上限になる。

---

## 反社会的行動ルール

壊す・殴る・罵倒などの反社会的行動は通常即罰金（大きなペナルティ）。
ただしそれがノルマだった場合、罰金は発生しない。

- 罰金あり → ノルマではない（損しただけ）
- 罰金なし → ノルマ確定（貴重な情報）

所持金に余裕があるときのハイリスク・ハイリターンな情報収集手段。
ドボンとは別レイヤー。

---

## ヒント供給源（複合）

| 手段 | 説明 |
|------|------|
| 常連行動の観察 | 常連の共通行動からノルマ候補を推理 |
| 盗み聞き | 席に座って常連のコソコソ話から裏メニュー名を知る |
| 環境ヒント | 店内の張り紙・小物から示唆を得る |
| 好感度アンロック | 達成率が高い来店を重ねると店主がヒントをくれる |
| 消去法 | ドボンを踏んだ結果、候補が絞れる |
| 反社会的行動の博打 | 罰金の有無でノルマを確定できる |

---

## 好感度システム

- 来店で少し上昇
- ノルマ達成数に応じて追加上昇
- ドボンで大幅下降
- 閾値を超えたら未達成ノルマ1つをランダムでアンロック

## プロトタイプスコープ

### 目的
コアループ「観察→模倣→判定」が面白いかを検証する。

### 含めるもの
- チュートリアル1ステージのみ
- 2Dサイドビュー、リアルタイム進行
- 操作ボタン4つ（左右移動、ジャンプ、定型文、インタラクト）
- メニューUI（画面占有 → 観察トレードオフ）
- 店主に話しかけて判定フェーズ
- 所持金・食事代（注文時即引き）・ドボン罰金（退店時）の基本コスト計算
- 盗み聞きシステム（席に座っていれば裏メニュー名が聞こえる）

### チュートリアルデータ

**客：常連A（1人のみ）**

| フェーズ | 行動 |
|----------|------|
| 注文 | 焼き魚定食、味噌汁、裏メニュー |
| 感想 | 「うまい」 |

**ノルマ**
1. 焼き魚定食を注文する
2. 「うまい」と言う

**ドボン**
- カレーを注文する

**選択肢プール**

| カテゴリ | 選択肢 |
|----------|--------|
| 注文 | 焼き魚定食 / 生姜焼き定食 / カレー（ドボン） / 味噌汁 / コーヒー |
| 感想 | 「うまい」/「量が多い」/「また来る」/「まずい」 |

**数値**

| 項目 | 値 |
|------|-----|
| 初期所持金 | 1500円 |
| メニュー単価 | 一律300円 |
| ドボン罰金 | 500円 |

### 含めないもの
- 好感度システム
- 環境ヒント
- ヒントアンロック
- 反社会的行動ルール
- 複数ステージ
- ジャンプ入店等の移動系ノルマ
- 演出全般
- 注文キャンセル
