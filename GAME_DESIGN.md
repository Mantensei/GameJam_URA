# GAME_DESIGN — 「裏メニューを食べたい！」

## 概要
Unity 2022.3.11f1 製の GameJam プロジェクト（WebGLビルド）。
常連客を観察し、裏メニューを注文するために必要な行動を模倣する推理アクションゲーム。

- ジャンル：観察→模倣→推理（リアルタイム進行）
- 視点：2Dサイドビュー
- ビジュアル：ドット絵
- クリア条件：裏メニューを注文する
- 敗北条件：所持金が0になる

---

## ゲーム構造

### 上位層：ステージ選択

```
ステージ一覧 → ステージ選択 → 来店（下位層へ） → 結果 → ステージ一覧に戻る
```

- クリア済みステージは裏メニューがコレクションに追加される
- 所持金が尽きたらゲームオーバー

### 下位層：1来店の流れ

```
入店 → 自由行動（移動・観察・模倣・注文・食事・感想） → 店主に話しかける → 判定 → 退店
```

1来店の中にフェーズ区切りはない。プレイヤーも客も自由に行動する。
時間はリアルタイムで進み、客は自身のタイムラインに沿って行動する。
1来店あたり2〜3分程度を想定。時間制限はない。

#### 入店
- 外と店内は地続き。入り口のドアが最前面レイヤーにあるだけ
- 来店時点で既に客がいる場合がある

#### 着席・注文・食事
- **注文：** 席に座ってメニューを開き、品を選ぶ。複数品注文可。即支払い。食事中に追加注文も可
- **食事：** 料理が届いたらインタラクトで食べる。食べるのには時間がかかる。複数品注文時は1品ずつ食べる

#### 感想・発言（定型文ボタン）
- 状況依存で選択肢が変わる（食事前後、場所、プレイヤーの状態）
- 押し得防止のためクールタイムあり

#### 店主に話しかける
- **食事後のみ可能**
- 話しかけると判定フェーズに入る

#### 退店
- 判定後は自動退店
- ドボン罰金はここでまとめて発生
- 結果画面が表示される

---

## 操作

ボタン4つ：

| ボタン | 用途 |
|--------|------|
| 左右移動 | 店内を移動、客や物に近づく |
| ジャンプ | 入店方法等（要検証、不要なら削除） |
| 定型文 | 感想などの発言系アクション（いつでも自由に使用可能） |
| インタラクト | 対象に応じた行動（席に座る/立つ、メニューを開く、食べる、店主に話しかける） |

## ノルマ・ドボン・判定

### ノルマ
- ステージごとに設定された「裏メニューを頼むための条件」
- 全て達成すると店主が裏メニューを教えてくれる
- 難易度が高いほど条件が増える（一部は達成しなくてもOKな場合あり）

### ドボン
- 実行すると罰金等のペナルティが発生する禁止行動
- 常連客がやらない行動を避ければ基本的には踏まない
- 難易度が高いほど増える
- 判定は退店時にまとめて行う

### 判定（店主に話しかけた時）
- 全ノルマ達成 → 「裏メニューを教えます」 → クリア
- 未達成 / 何もしていない → 「帰んなさい」 → 自動退店
- ドボンがあった場合は退店時に罰金が引かれる

---

## 客

### 常連客
- 裏メニューを知っている客
- メニュー表にない品（裏メニュー）を注文する → 常連の識別手段
- 行動の中にノルマのヒントが含まれる
- ただし常連の行動が全てノルマとは限らない（ダミーあり）
- Max 4人

### 一般客
- メニュー表にある品だけを注文する
- 行動にはドボンに繋がるものが含まれる場合がある

### 注文の公開
- 全ての客の注文内容はプレイヤーに公開される

### 行動タイムライン
全ての客は以下の基本フローで行動する（リアルタイム進行）：

```
入店 → 着席 → [注文 → 食事]（繰り返し可） → 全品食べ終わり → 退店
```

- 入店タイミングはランダム（プレイヤー来店時に既に座っている客もいる）
- 注文は何回でも可能（一度に複数品、食後の追加注文も可）
- 出された料理を全て食べきるまで完全退店できない
- メインフローの「退店」は来店の完了を意味する
- 割り込み行動はメインフローの進行とは無関係にいつでも発生する：
  - 感想は常時発言可能（クールタイムあり）
  - 着席・離籍は自由（食事中でも席を離れられる）
  - 食事をキャンセルして別の席で食べ直すことがある
  - 一時外出（店の外に出て戻ってくる）、トイレ等

---

## 反社会的行動ルール

壊す・殴る・罵倒などの反社会的行動は通常即罰金（大きなペナルティ）。
ただしそれがノルマだった場合、罰金は発生しない。

- 罰金あり → ノルマではない（損しただけ）
- 罰金なし → ノルマ確定（貴重な情報）

所持金に余裕があるときのハイリスク・ハイリターンな情報収集手段。
ドボンとは別レイヤー。

---

## ヒント供給源（複合）

| 手段 | 説明 |
|------|------|
| 常連行動の観察 | 常連の共通行動からノルマ候補を推理 |
| 環境ヒント | 店内の張り紙・小物から示唆を得る |
| 好感度アンロック | 達成率が高い来店を重ねると店主がヒントをくれる |
| 消去法 | ドボンを踏んだ結果、候補が絞れる |
| 反社会的行動の博打 | 罰金の有無でノルマを確定できる |

---

## 好感度システム（低優先度）

- 来店で少し上昇
- ノルマ達成数に応じて追加上昇
- ドボンで大幅下降
- 閾値を超えたら未達成ノルマ1つをランダムでアンロック
